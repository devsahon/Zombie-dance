HOW TO USE
================================================================================

STEP 1: SETUP DATABASE
   mysql -u root -p < server/database/schema.sql

STEP 2: POPULATE DEMO DATA
   cd server && npm run populate-demo-data && cd ..

STEP 3: CONFIGURE ENVIRONMENT
   cp .env.example .env.local
   # Edit .env.local with your database credentials

STEP 4: START SERVICES
   # Terminal 1:
   cd server && npm run dev
   
   # Terminal 2:
   npm run dev

STEP 5: VERIFY
   curl http://localhost:8000/health
   curl http://localhost:8000/models
   # Visit http://localhost:3000

TROUBLESHOOTING
================================================================================

If database won't connect:
→ Verify MySQL is running
→ Check credentials in .env
→ Ensure database exists: mysql -u root -p -e "SELECT 1 FROM uas_admin.ai_models;"

If demo data won't populate:
→ Ensure schema is created first
→ Check file path is correct
→ Verify permissions
→ Check error logs

If API returns 500 error:
→ Check server console for error messages
→ Verify database connection
→ Review error handling middleware

If CORS errors appear:
→ Check CORS_ORIGIN in server/.env
→ Verify URLs match configuration
→ Restart backend server
